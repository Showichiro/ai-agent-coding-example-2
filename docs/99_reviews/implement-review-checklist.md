
# 実装レビューチェックリスト

## コーディング規約

- [ ] 規約違反はないか（リンター、フォーマッター）
- [ ] 命名は適切か（変数名、関数名、クラス名など）

## 設計

- [ ] 設計書通りに実装されているか
- [ ] 単一責任の原則が守られているか
- [ ] 凝集度は高く、結合度は低く保たれているか

## 可読性

- [ ] コードは読みやすいか
- [ ] コメントは適切か（不要なコメント、不足しているコメントはないか）

## テスト

- [ ] テストは十分か（カバレッジ、テストケース）
- [ ] テストは意図通りに動作しているか
